## S.M.A.R.T. задачи (MVP, 3 месяца)

### Этап 0. Инфраструктура ✅

- Specific: развернуть облачную инфраструктуру в Yandex Cloud (Terraform): S3-бакет, VPC, DataProc кластер (Spark).
- Measurable: Terraform-конфигурация в репозитории, данные скопированы в S3, доступ к кластеру по SSH.
- Status: **Выполнено** (ДЗ №2). S3-бакет создан, данные загружены, кластер разворачивается по `make apply`.

### Этап 1. Подготовка данных и EDA ✅ / 🔄

- Specific: провести анализ качества данных, выявить проблемы, автоматизировать очистку; подготовить датасет для обучения.
- Measurable: ≥3 типа проблем найдено (найдено 6), скрипт очистки `data_cleaning.py`, данные в Parquet.
- Status: **Выполнено** (ДЗ №3). Найдено 6 типов проблем, создан ноутбук анализа и PySpark-скрипт очистки.
- Осталось: Feature engineering (Feast — ДЗ №4), time-split для train/valid/test.

### Этап 2. Feature Store (Feast)

- Specific: создать Feature Views для агрегатных фичей (частота транзакций, средний чек, поведенческие паттерны), on-demand фичи для инференса.
- Measurable: ≥2 Feature View + 1 on-demand, ноутбук с примерами получения фичей.
- Achievable: 2 недели.
- Relevant: переиспользуемые фичи ускоряют итерации по моделям.
- Time-bound: 2 недели (ДЗ №4).

### Этап 3. Автоматизация пайплайна (Airflow)

- Specific: развернуть Apache Airflow в Yandex Cloud, создать DAG для периодической очистки новых данных (ежедневно: создание Spark-кластера → очистка → удаление).
- Measurable: DAG в Airflow UI, ≥3 успешных запуска, скриншоты.
- Achievable: 2 недели.
- Relevant: без автоматизации очистка данных — ручной процесс.
- Time-bound: 2 недели (ДЗ №5).

### Этап 4. Обучение модели и MLFlow

- Specific: обучить модель (PySpark), логировать метрики и артефакты в MLFlow, настроить периодическое переобучение через Airflow.
- Measurable: PR-AUC ≥ 0.90, Recall@FPR≤5% ≥ 0.75, модель в MLFlow registry, DAG переобучения.
- Achievable: 2 недели.
- Relevant: достигает бизнес-ограничения по FP.
- Time-bound: 2 недели (ДЗ №6).

### Этап 5. Онлайн-скоринг сервис (MVP)

- Specific: REST API сервис со скорингом, p95 ≤ 150 мс при 400 RPS; логирование решений (без PII), фича-кеш.
- Measurable: нагрузочный отчёт, дашборд latency/error rate.
- Achievable: 3 недели.
- Relevant: реалтайм-проверка транзакций — ключевой сценарий.
- Time-bound: 3 недели.

### Этап 6. Мониторинг качества и дрейфа

- Specific: мониторинг данных/скорингов (Evidently), алерты по drift и падению Recall@FPR≤5%.
- Measurable: алерты в Slack/Email, дашборды, run-книги инцидентов.
- Achievable: 2 недели.
- Relevant: предотвращает деградацию.
- Time-bound: 2 недели.

### Этап 7. Безопасность и комплаенс PII

- Specific: внедрить псевдонимизацию, шифрование, секрет-менеджмент, RBAC; ревью логов на отсутствие PII.
- Measurable: чек-лист комплаенса, политики KMS, IaC для секретов.
- Achievable: 2 недели, параллельно с другими этапами.
- Relevant: критично для бизнеса.
- Time-bound: 2 недели.
